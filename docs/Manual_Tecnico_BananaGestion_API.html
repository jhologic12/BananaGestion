1. Introducción

Base URL: http://localhost:5000
Swagger UI: http://localhost:5000/swagger
Formato de respuesta: JSON

Roles del Sistema

Rol Código Descripción Permisos
Administrador Administrador Control total del sistema Todas las operaciones CRUD
Supervisor Supervisor Gestión operativa Crear, leer, actualizar tareas y cosechas
Obrero Obrero Ejecución de tareas Ver y completar tareas asignadas

Encabezados Requeridos

Encabezado Valor Descripción
Content-Type application/json Para peticiones con body JSON
Authorization Bearer {token} Token JWT para endpoints protegidos

2. Autenticación

POST /api/Auth/login

Iniciar sesión en el sistema y obtener token JWT.

Request Body

{
  "email": "usuario@ejemplo.com",
  "password": "contraseña123"
}

Response (200 OK)

{
  "id": "uuid-del-usuario",
  "email": "usuario@ejemplo.com",
  "nombre": "Juan",
  "apellido": "Pérez",
  "rol": "Administrador",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
POST /api/Auth/register

Registrar un nuevo usuario en el sistema.

Request Body

{
  "email": "nuevo@ejemplo.com",
  "password": "minimo6caracteres",
  "nombre": "Nombre",
  "apellido": "Apellido",
  "telefono": "555-1234",
  "rol": "Administrador" // Administrador | Supervisor | Obrero
}

Response (200 OK)

{
  "id": "uuid-generado",
  "email": "nuevo@ejemplo.com",
  "nombre": "Nombre",
  "apellido": "Apellido",
  "rol": "Administrador",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
POST /api/Auth/change-password [Autenticado]

Cambiar la contraseña del usuario autenticado.

Headers

Authorization: Bearer {token}

Request Body

{
  "currentPassword": "contraseñaActual",
  "newPassword": "nuevaContraseña123"
}

3. Gestión de Usuarios

GET /api/Users [Administrador, Supervisor]

Obtener lista de todos los usuarios.

Response

[{
  "id": "uuid",
  "email": "usuario@ejemplo.com",
  "nombre": "Juan",
  "apellido": "Pérez",
  "telefono": "555-1234",
  "rol": "Supervisor",
  "activo": true,
  "fechaCreacion": "2024-01-15T10:30:00Z",
  "ultimoLogin": "2024-01-20T08:00:00Z"
}]
GET /api/Users/{id}

Obtener usuario por ID.

GET /api/Users/obreros

Obtener lista de usuarios con rol Obrero.

GET /api/Users/supervisores

Obtener lista de usuarios con rol Supervisor.

PUT /api/Users/{id}

Actualizar datos del usuario.

Request Body

{
  "nombre": "Nuevo Nombre",
  "apellido": "Nuevo Apellido",
  "telefono": "555-5678"
}
DELETE /api/Users/{id} [Administrador]

Eliminar usuario (soft delete).

8. Códigos de Error

Código HTTP Descripción
200 Operación exitosa
201 Recurso creado exitosamente
400 Bad Request - Datos inválidos o faltantes
401 Unauthorized - Token inválido o expirado
403 Forbidden - No tiene permisos para esta acción
404 Not Found - Recurso no encontrado
409 Conflict - Recurso duplicado
500 Internal Server Error - Error del servidor

Formato de Error

{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "Bad Request",
  "status": 400,
  "detail": "La solicitud no pudo ser procesada",
  "errors": {
    "Email": ["El campo Email es requerido"]
  }
}
